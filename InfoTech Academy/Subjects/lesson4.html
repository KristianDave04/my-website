    IT214 - Cloud Computing Basics

    The Basics of Cloud Computing


    What is Cloud Computing?
    Cloud computing is a widely used, yet often misunderstood concept. At its core, it’s a subscription-based service that provides access to shared computing resources—such as storage and applications over the internet.

    To understand it, consider your web-based email. Services like Gmail or Yahoo! Mail host the necessary infrastructure, letting you access your emails from any device with an internet connection. Unlike software installed directly on your device, your email is stored remotely, offering flexibility and convenience. Cloud computing applies this model to a wide range of data and applications, not just email.


    How do you use Cloud?
    The cloud allows you to access your data and applications from anywhere with an internet connection. It eliminates the need to be physically near your storage devices or computing infrastructure.

    This flexibility is especially beneficial for small businesses, which can avoid the cost of purchasing and maintaining hardware. Instead, they can rent the exact amount of storage and computing power they need and scale up or down as necessary.

    All that’s required is an internet connection—wired, wireless, or mobile. Whether you're using a phone, tablet, or computer, the cloud makes it easier to collaborate, access documents, or run applications remotely.


    Types of Cloud Deployment

    Depending on your needs, there are four main types of cloud environment.

    1. Public Cloud – Open to any subscriber with internet access
    2. Private Cloud – Restricted to a specific organization or group
    3. Community Cloud – Shared by organizations with similar needs
    4. Hybrid Cloud – Combines two or more cloud types (e.g., public and private).

    Choosing a Cloud Provider

    Selecting a provider depends on your specific requirements. Cloud services fall into three main categories, each offering different levels of control.

    1. Software as a Service (SaaS): Provides access to applications over the internet without installing them on local devices. You have the least control but benefit from ease of use.
    2. Platform as a Service (PaaS): Offers tools for developing and deploying applications online, Suitable for developers who want more control than SaaS offers.
    3. *Infrastructure as a Service (IaaS): Delivers virtualized hardware resources. This offersthe most control, ideal for businesses needing to manage their computing environment directly.

    As you move from SaaS to IaaS, user control increases while the provider’s role decreases.


    Security Considerations

    Security is one of the most critical concerns in cloud computing. Your data, now hosted offsite, becomes a potential target for cyber threats. You must evaluate your provider's security policies, including:

    • Data encryption methods
    • Physical security of servers
    • Backup procedures
    • Firewall protections
    • Isolation safeguards in shared environments (e.g., community clouds)

    Home users typically accept default provider settings, while businesses may negotiate security terms. Regardless of your role, selecting a provider that prioritizes data protection is essential.

    Remember: using the cloud means relinquishing some control over your data's physical location and security, but most providers are better equipped to safeguard data than individual users.

    Cloud computing opens new possibilities for individuals and businesses alike. It provides scalable, flexible, and cost-effective access to computing resources. However, it also requires careful consideration of data control, access, and security risks.

    Before moving to the cloud, assess:

    • What data you'll store
    • Who will have access
    • The level of control and services you need
    • The reputation and policies of your cloud provider

    With thoughtful planning, the cloud can be a powerful tool to enhance productivity and efficiency.


    IaaS, PaaS, SaaS Models


    Introduction to Cloud Models
    Cloud computing enables developers and IT teams to focus on core business goals by outsourcing infrastructure responsibilities such as procurement, maintenance, and scaling. The three primary cloud service models—Infrastructure as a Service (IaaS), Platform as a Service (PaaS), and Software as a Service (SaaS)—offer varying levels of control and management.

    Understanding these models and how AWS delivers each helps businesses select the most suitable services to meet their needs.

    Introduction to AWS Cloud

    Amazon Web Services (AWS) provides a comprehensive suite of cloud solutions aligned with business goals. AWS simplifies global collaboration, operations, and infrastructure management using frameworks like the AWS Cloud Adoption Framework and the AWS Hybrid Cloud Framework.

    AWS helps businesses streamline development, reduce infrastructure investment, and scale efficiently—serving customers across over 190 countries.

    Cloud Model Overview: SaaS, PaaS, IaaS

    Service Model Managed by You Managed by Provider
    IaaS Applications, Data, Runtime, Middleware, OS Virtualization, Servers, Storage, Networking
    PaaS Applications, Data Runtime, Middleware, OS, Infrastructure
    SaaS Just use the app Everything managed by provider

    AWS as an IaaS Provider

    AWS leads the IaaS market with services like Amazon EC2, which provides secure, scalable, and cost-effective compute capacity. Benefits include:

    • Full control of virtual machines
    • Scalability with auto-scaling and load balancing
    • High availability with 66+ Availability Zones across 21+ regions
    • Advanced security via AWS Identity and Access Management (IAM), Multi-Factor Authentication (MFA), and AWS Key Management Service (KMS)

    Key IaaS services:

    • Amazon VPC for private networking
    • AWS Elasticsearch for load balancing and operational efficiency
    • Amazon RDS for high-availability data replication
    • Amazon Cognito for secure authentication

    IaaS suits businesses needing full control over applications and infrastructure without managing physical hardware.

    AWS as a SaaS Provider

    SaaS allows organizations to use software hosted and maintained by a provider, with no infrastructure or platform concerns.


    Why businesses adopt SaaS via AWS:

    • No need for hardware or installations
    • Immediate scalability and flexibility
    • Secure access from anywhere
    • Pay-as-you-use pricing
    • Integrated backup, monitoring and analytics tools

    Challenges of SaaS adoption:

    • Limited customization
    • Integration complexity with existing on-premise systems.
    • Dependence on provider for performance and uptime
    • Data Security and compliance concerns

    AWS SaaS capabilities:

    • Secure, multi-tenant architecture
    • Identity Management with IAM
    • Real-time monitoring with Amazon CloudWatch
    • Backup management with AWS Backup
    • Analytics with Amazon Kinesis

    SaaS on AWS is ideal for businesses seeking fast, low-maintenance software deployment.


    AWS as a PaaS Provider
    AWS PaaS solutions such as AWS Lambda abstract infrastructure management, letting developers focus solely on code.

    What AWS Lambda offers:

    • Event-driven, serverless architecture
    • Pay only for compute time used
    • Scales automatically with demand
    • Supports integration with other AWS services like AI/ML

    Benefits of AWS PaaS:

    • Eliminates infrastructure overhead
    • Speeds up development cycles
    • Ensures secure, isolated execution environments
    • Facilitates real-time event handling and automation

    AWS PaaS is ideal for agile teams needing scalable environments for application development without hardware management.

    Virtual Machines: Computers Within Computers


    What is a Virtual MAchine?

    A Virtual Machine allows you to emulate an entire computer system within another. This setup enables software to run as if it were on its own dedicated hardware.

    Key Components:

    • Hypervisor: Manages the VMs
    • Host: The real physical computer running the hypervisor
    • GuestL The virtualized system running with in the host

    Why Use Virtual Machines?

    Primary Use Cases:

    • Isolation & Sandboxing: Run untrusted code safely
    • Resource Allocation: Control CPU, memory, and storage usage
    • System Management: Easy to backup, restore, or restart environments.
    • Cross-Platform Testing: Test software across OSes
    • Prototyping NetworkdsL Simulate full networks (e.g., with GNS3)
    • Emulation: Run legacy software for different CPU architectures
    • Experimentation: Try Linux or other OSes without altering your main system

    Emulation vs Virtualization

    • Emulation: Simulates a completely different hardware platform
    • Virtualization: Shares the same architecture as the host and is more efficient

    Virtualization is faster because it uses the host’s hardware directly, whereas emulation recreates hardware in software.


    Virtualization & Security

    • VMs are designed to isolate the guest from the host
    • Ideal for testing destructive commands(e.g., rm -rf/) safely
    • CPUs include features like intel VT-x and AMD-V to optimize virtualization.

    Hypervisors: The Backbone of VMsTwo Types:

    1. Type 1 (Bare MEtal)-Hypervisor is the OS
      • Examples:VMWare ESXi, Hyper-V, Promox, Xen
    2. Type 2 (Hosted)-Hypervisor runs inside another OS
      • Examples: VirtualBox, UTM, VMWare Workstation, Parallels

    Container:Lightweight Virtualization

    What are Containers?

    Containers package applications with their dependencies but share the host OS kernel. They are more lightweight than VMs and are designed for:

    • Speed
    • Efficiency
    • Ephemerality(temporary by default; persistent data is stored in volumes)

    Containerization: Consistent Environments

    Containers define:

    • Expected OS environment (not the kernel)
    • CPU architecture
    • Application dependencies
    • Filesystem layout

    They offer isolated execution environments for consistency across systems.

    Docker: Containerization in Action

    Key Docker Concepts:

    • Image: A snapshot of an app with its dependencies
    • Container: A running instance of an image
    • Dockerfile: Script to build an image
    • Volume: Stores persistent data outside the container lifecycle

    Docker Analogy:

    • Image: Like a .zip or .dmg file
    • Container: The extracted app
    • Dockerfile: The recipe for building the image
    • Volume: Where the app stores user data

    Docker Example:

    Bash.

    docker run -it ubuntu:latest bash

    • i keep STDIN open
    • t allocates a pseudo-TTY
    • ubuntu:latest is the image name
    • bash is the command to run

    Building with Dockerfiles

    Example Dockerfile:

    FROM ubuntu:latest
    RUN apt-get -y update && apt-get -y upgrade
    RUN apt-get install -y build-essential
    WORKDIR /calculator_app
    ADD ./calculator/ ./
    RUN make clean && make
    ENTRYPOINT ["./calculator"]


    Key Elements:

    • FROM: Base image
    • RUN: Executes commands to prepare the image
    • WORKDIR: Sets the working directory
    • ADD: Copies files into the image
    • ENTRYPOINT: Defines the startup command

    Scaling with Orchestration

    When managin many containers, orchestration tools are used to automate deployment, scaling, and networking.

    Kubernetes(KBs):

    • High-level management for containers
    • Handles load balancing, resource allocation, and autoscaling
    • Best for large-scale deployments across multiple servers
    • Commonly used in cloud environments (e.g., AWS, GCP, Azure)

    Deploying Applications in the Cloud

    Cloud providers offer multiple options:

    • VPS (Virtual Private Server): A VM you control
    • App Services: Provider runs your containerized apps
    • Kubernetes Services: Cloud-integrated K8s deployments

    Conclusion:

    • VMs: Full system emulation, ideal for isolation and testing
    • Containers: Lightweight, efficient, ideal for consistent environments
    • Docker: The de facto standard for managing containers
    • Kubernetes: Powerful orchestration for scaling containerized apps

    Security in the Cloud


    Cloud computing has revolutionized the way businesses and individuals manage data and deploy applications, but as more sensitive information is stored and processed remotely, the security of these cloud environments has become a critical concern. Security in the cloud involves protecting data, applications, and systems from threats while maintaining availability, confidentiality, and integrity in a shared infrastructure.

    In this detailed overview, we will examine cloud security principles, threats, and best practices for securing cloud environments.


    1. Key Principles of Cloud Security

      2. Confidentiality

      Confidentiality ensures that data stored in the cloud is only accessible to authorized users. This is crucial for organizations that manage sensitive customer or proprietary information.

      • Data Encryption: Encrypting data at rest (when stored on the cloud) and in transit (when transmitted over networks) is a fundamental aspect of ensuring confidentiality.
      • Access Control: Using strong identity and access management (IAM) protocols ensures that only authorized users can access cloud resources. Techniques such as multi-factor authentication (MFA) add an additional layer of security.

      Integrity

      Integrity ensures that data is not tampered with and that it remains accurate and trustworthy. Cloud providers must ensure that their systems are free from unauthorized changes.


      • Checksums and Hashing: Data integrity is verified using checksums or hash algorithms, which generate a unique value for the data that can be checked to detect alterations.
      • Version Control and Audit Trails: Maintaining a clear version history and audit logs for cloud resources can help track changes and identify any potential unauthorized modifications.

      Availability

      Availability refers to ensuring that cloud resources are accessible and operational when needed. This is especially important for businesses relying on cloud applications to operate continuously.


      • Redundancy and Failover Systems: To ensure availability, cloud providers use redundant systems and data replication across different geographic regions. This ensures that in case of hardware failure, services remain available.
      • Service Level Agreements (SLAs): SLAs outline the expected uptime and availability guarantees. These agreements help organizations understand the levels of service they can expect from the cloud provider.

    2. Common Cloud Security Risks

      3. Data Breaches

      A data breach occurs when unauthorized individuals gain access to sensitive data. With the growing amount of personal and financial data stored in the cloud, breaches have become one of the biggest risks.


      • Insider Threats: Employees or contractors with malicious intent or those who accidentally mishandle data can pose a significant risk to cloud security.
      • Hacking and Phishing Attacks: Hackers use advanced techniques like spearphishing, malware, and ransomware to breach cloud services.

      Loss of Control Over Data

      In the cloud, data is stored and managed by third-party providers. This can lead to concerns about data ownership and control, especially if data is stored across borders with varying laws and regulations.


      • Jurisdictional Concerns: Data may be subject to different privacy laws depending on where it is stored, complicating compliance with regulations like the GDPR or CCPA.
      • Vendor Lock-In: Organizations may become overly dependent on a particular cloud service, making it difficult to migrate to other platforms or to regain control of their data.

      Misconfiguration of cloud Resources

      Many cloud security incidents stem from improper configuration of cloud resources, often due to lack of experience or misunderstanding of security tools available in the cloud.


      • Open Storage Buckets: Public cloud storage buckets (e.g., Amazon S3) are sometimes left open, exposing sensitive data to anyone on the internet.
      • Improper Network Security Settings: Misconfigured firewalls, security groups, or access control lists (ACLs) can create vulnerabilities in cloud infrastructure.

      Denial of service (DoS) Attacks

      DoS attacks target the availability of cloud services by overwhelming them with traffic, making them inaccessible to legitimate users.


      • Distributed Denial of Service (DDoS): A DDoS attack amplifies the effect by using multiple systems to flood the target, often making it difficult to mitigate.

    3. Cloud Security Frameworks and Standards

      4. Several frameworks and industry standards exist to guide organizations in securing their cloud environments and ensuring compliance with relevant regulations.

      ISO/IEC 27001

      This international standard provides a systematic approach to managing sensitive company information, including risk management, and sets the foundation for cloud security practices.

      NIST Cybersecurity Framework

      The National Institute of Standards and Technology (NIST) has developed a comprehensive cybersecurity framework, which includes cloud-specific controls for securing cloud computing environments.

      • NIST 800-53: A publication offering a catalog of security and privacy controls for federal information systems and organizations.

      CSA Cloud Controls Matrix (CCM) The Cloud Security Alliance (CSA) provides a set of security controls specifically designed for cloud environments. The CCM addresses key security concerns like data security, identity and access management, and infrastructure security


    4. Best Practices for Cloud Security

      5. Data Encryption

      Encrypt sensitive data both at rest and in transit. Cloud service providers often offer tools to manage encryption, but it’s essential that organizations implement their encryption strategies, ensuring only authorized users can access data.

      Identity and Access Management (IAM)

      IAM is critical to ensuring that only authorized users have access to cloud resources.

      • Least Privilege Principle: Users and services should only have the minimum level of access necessary to perform their tasks.
      • Multi-Factor Authentication (MFA): MFA reduces the risk of unauthorized access by requiring more than just a password to authenticate users.

      Regular Audits and Monitoring

      Implement continuous monitoring of cloud resources, focusing on unusual activity, changes in configurations, or access patterns.

      • Log Management: Cloud providers may offer logging solutions, but integrating those with your own security information and event management (SIEM) system ensures a comprehensive view of potential threats.

      Compliance and Regulatory Adherence

      Ensure that your cloud provider and services comply with relevant regulatory frameworks such as GDPR, HIPAA, PCI-DSS, or SOX.

      • Data Sovereignty: Understand where data is stored geographically, and ensure that the location meets legal and regulatory requirements.

      Backup and Disaster Recovery Plans Maintaining regular backups of critical data is important to ensure continuity in case of a security incident. Implement disaster recovery (DR) plans to minimize downtime in the event of an attack or system failure.

      Network Security and Firewalls

      Use virtual private networks (VPNs) and firewalls to secure cloud-based systems from external threats. Configure security groups and network ACLs carefully to control the flow of traffic to and from cloud resources.

      Employee Education and Training Cloud security often begins with the users. Regular training helps employees recognize phishing attacks, use strong passwords, and follow best practices when interacting with cloud systems.


    5. Emerging Technologies and Cloud Security trends

      6. Zero Trust Architecture Zero Trust assumes that no user or device, whether inside or outside the network, is trustworthy by default. Every request to access cloud resources must be authenticated and authorized.

      • Micro-Segmentation: Zero Trust often involves segmenting cloud resources into smaller, isolated zones to reduce the potential impact of security breaches.

        • AI and Machine Learning for Threat Detection

        Artificial Intelligence (AI) and machine learning (ML) are increasingly being used to detect unusual patterns of behavior and potential threats in cloud environments.

        • Anomaly Detection: These technologies can quickly identify abnormal activity, such as unauthorized access or data exfiltration.

        Serverless Security

        Serverless computing abstracts infrastructure management, and while this improves scalability, it also introduces new security challenges, including securing API endpoints and handling vulnerabilities in shared environments.
