    UNIT III: Routing and Switching

    1. What is Routing and Switching?
    * Routing refers to the process of selecting paths in a network along which to send data packets from source to destination.
    * Switching involves forwarding data within a local network (LAN) using MAC addresses.
    * Routing operates at Layer 3 (Network Layer) of the OSI model, while switching operates at Layer 2 (Data Link Layer).

    2. Switching Concepts

    A. What is a Switch?
    A switch is a Layer 2 device that connects multiple devices within the same network and uses MAC addresses to forward data only to the intended recipient.

    B. Switch Functions
    * Learns MAC addresses via the MAC address table.
    * Forwards frames based on destination MAC.
    * Reduces unnecessary traffic compared to a hub.
    * Operates in full-duplex mode.

    C. Types of Switching
    * Store-and-Forward: Entire frame is received before forwarding.
    * Cut-Through: Starts forwarding as soon as destination MAC is read.
    * Fragment-Free: Checks the first 64 bytes for errors before forwarding.

    D. VLANs (Virtual LANs)
    * Used to segment broadcast domains in a switch.
    * Each VLAN behaves like a separate logical network.
    * Increases security and reduces broadcast traffic.

    3. Routing Concepts

    A. What is a Router?
    A router is a Layer 3 device that connects different networks and determines the best path for data to travel across networks using IP addresses.

    B. Key Routing Function
    * Maintains a routing table with paths to various networks.
    * Forwards packets based on destination IP.
    * Can perform Network Address Translation (NAT) and firewall filtering.

    C. Types of Routing

    Type Description Example
    Static Routing Manually configured routes Small networks
    Dynamic Routing Routes learned from routing protocols Larger, complex networks

    D. Routing Protocols

    Protocol Description Metric Used In
    RIP (Routing Information Protocol) Uses hop count; older Hop count •Small networks
    OSPF (Open Shortest Path First) Link-state protocol; efficient Cost •Enterprise networks
    EIGRP (Cisco proprietary) Advanced distance-vector Bandwidth, delay •Cisco networks
    BGP (Border Gateway Protocol) Used on the internet Path attributes •ISPs, Internet

    Feature Switch Router
    OSI Layer Layer 2 Layer 3
    Uses Address MAC Address IP Address
    Role Connects devices in LAN Connects different networks
    Speed Faster (no complex decisions) Slower (routing logic involved)
    Broadcast Domain Divides using VLANs Each port is a separate domain

    5. Packet Forwarding Flow
    Example: Data from PC A to PC B in another network
    1. Switch sends data to router via MAC address.
    2. Router checks IP and routing table.
    3. Router forwards packet to destination router.
    4. Final switch sends data to PC B.

    6. Routing and Switching Tools
    * Cisco Packet Tracer: Simulate switching and routing environments.
    * Wireshark: Monitor packet forwarding and routing paths.
    * Traceroute: Command-line tool to track the path of packets.

    7. Challenges in Real Networks
    * Loops (solved by Spanning Tree Protocol)
    * Congestion (solved by QoS and load balancing)
    * Security (handled via ACLs and VLAN segmentation)
