    UNIT II: OSI and TCP/IP Models

    1. Purpose of Network Models
    Network models are conceptual frameworks used to understand and design how data is transmitted across networks. They standardize communication functions to enable interoperability among different systems and devices

    A. OSI Model (Open Systems Interconnection Model)
    The OSI model has 7 layers, each with a distinct function.

    Name Application Presentation Session Transport Network Data Link Physical
    Description Interface for end-user processes. Data translation, encryption, compression Manages sessions and controls dialogues Ensures complete data transfer with error checking and flow control. Handles logical addressing and routing. Provides physical addressing (MAC), error detection. Transmits raw bit stream over physical medium.
    Protocols/Examples HTTP, FTP, SMTP JPEG, SSL, ASCII NetBIOS, PPTP TCP, UDP IP,ICMP,IPsec Ethernet, PPP, ARP Cables, Hubs, Fiber

    Layer Functions Summary:
    * Layers 1–4: Deal with *data transport.
    * Layers 5–7: Deal with *application-level interactions.

    Data Flow
    As data travels from sender to receiver, each layer adds or removes headers relevant to its role – a process called encapsulation and decapsulation.

    B. TCP/IP Model (Internet Protocol Suite) The TCP/IP model is a more practical and widely used model. It has 4 layers, which correspond loosely to the OSI model.

    Layer Description OSI Equivalent Protocols
    4 Application OSI Layers 5-7 HTTP, FTP,DNS, SMTP
    3 Transport OSI Layer 4 TCP,UDP
    2 Internet OSI Layer 3 IP, ICMP
    1 Network Access OSI Layer 1-2 Ethernet, Wi-Fi

    Key Differences

    OSI Model TCP/IP Model
    7 layers 4 layers
    Theoretical model Practical implementation
    Developed by ISO Developed by DARPA/DoD
    More detailed More simplified and real-world applicable

    Common Protocols Explained
    * HTTP (HyperText Transfer Protocol): Transfers web pages.
    * FTP (File Transfer Protocol): Transfers files between systems.
    * TCP (Transmission Control Protocol): Reliable, connection-based communication.
    * UDP (User Datagram Protocol): Faster, connectionless communication.
    * IP (Internet Protocol): Provides logical addressing and routing.
    * DNS (Domain Name System): Resolves domain names to IP addresses

    Encapsulation Example
    1. Application Layer: Creates a message (e.g., an email).
    2. Transport Layer: Breaks data into segments and adds port numbers.
    3. Internet Layer: Adds IP address to each segment.
    4. Network Access Layer: Adds MAC address and prepares the frame for transmission.

    IP Addressing and Subnetting
    1. What is an IP Address?
    An IP addres (Internet Protocol address) is a unique identifier assigned to each device on a network to enable communication.

    * IPv4 Format: Four decimal numbers separated by dots (e.g., `192.168.1.1`)
    * IPv6 Format: Eight groups of four hexadecimal digits (e.g.,
    `2001:0db8:85a3:0000:0000:8a2e:0370:7334`)

    2. Types of IP Addresses (IPv4)

    Type Description Example
    Private IP Used within local networks `192.168.x.x`, `10.x.x.x`
    Public IP Routable on the internet Assigned by ISP
    Static IP Manually configured and Unchanging. Common in servers
    Dynamic IP Assigned by DHCP, can change. Used by most home devices
    Loopback IP Used for internal testing `127.0.0.1`
    APIPA Auto-assigned if DHCP fails `169.254.x.x`

    3. Classes of IPv4 Addresses

    Class Range Default Subnet Mask Use Case
    A 1.0.0.0 – 126.0.0.0 255.0.0.0 Large networks
    B 128.0.0.0 – 191.255.0.0 255.255.0.0 Medium networks
    C 192.0.0.0 – 223.255.255.0 255.255.255.0 Small networks
    D 224.0.0.0 – 239.255.255.255 - Multicast
    E 240.0.0.0 – 255.255.255.255 - Experimental
    • Note: 127.x.x.x is reserved for loopback testing.

    4. Subnetting Basics
    -Subnetting is the process of dividing a larger network into smaller, manageable segments (subnets).
    -It improves performance and enhances security.

    Key Concepts:

    • Subnet Mask: Determines the network and host portion of the IP address.

    * Example: `255.255.255.0`
    * CIDR Notation: Represents subnet using slash format (e.g., `192.168.1.0/24`)
    * Network Address: First address in the subnet (e.g., `192.168.1.0`)
    * Broadcast Address: Last address in the subnet (e.g., `192.168.1.255`)
    * Usable IP Range: Between network and broadcast address (e.g., `192.168.1.1 – 192.168.1.254`)

    5. Calculating Subnets
    Example:
    Given: IP – `192.168.1.0/26`
    * Subnet mask: `255.255.255.192`
    * Number of subnets: 4
    * Hosts per subnet: 62 usable (64 total – 2 for network and broadcast)
    * Subnets:
    * `192.168.1.0/26` → Hosts: `192.168.1.1 – 192.168.1.62`
    * `192.168.1.64/26` → Hosts: `192.168.1.65 – 192.168.1.126`
    * etc.

    Formulae:
    * Number of Subnets = 2^n (n = number of borrowed bits)
    * Number of Hosts = 2^h – 2 (h = number of host bits left)

    6. Binary Notation and Subnetting
    Understanding binary is key to subnetting:
    * Decimal: `192` → Binary: `11000000`
    * IP: `192.168.1.0` → Binary: `11000000.10101000.00000001.00000000`
    * Subnet Mask: `255.255.255.0` → Binary: `11111111.11111111.11111111.00000000`

    7. IPv6 Overview (Introductory Level)
    * 128-bit addresses (vs 32-bit in IPv4)
    * Written in hexadecimal, separated by colons
    * Designed to solve IPv4 exhaustion
    * Example: `2001:0db8:85a3:0000:0000:8a2e:0370:7334`
