    UNIT VI: Network Security Basics

    1. What is Network Security?
    • Network security refers to the policies, practices, and technologies used to protect network infrastructure, data, and devices from unauthorized access, misuse, malfunction, or destruction.
    • It aims to ensure the **confidentiality, integrity, and availability (CIA)** of data transmitted across or stored within networks.

    2. Goals of Network Security

    Security Goal Description
    Confidentiality Prevent unauthorized access to data (e.g., encryption)
    Integrity Ensure data is not tampered with (e.g., hashing)
    Availability Keep services and data accessible when needed (e.g., DDoS protection)
    Authentication Verify identity of users and systems
    Non-repudiation Ensure actions can’t be denied later (e.g., digital signatures)

    3.Common Threats to Networks

    Threat Type Description
    Malware Viruses, worms, ransomware, trojans
    Phishing Deceptive emails or websites to steal information
    Denial of Service (DoS/DDoS) Overloads the network to make it unavailable
    Man-in-the-Middle (MitM) Intercepts communication between two parties
    IP Spoofing Faking an IP address to gain access
    Eavesdropping Monitoring network traffic to steal data
    Brute-force attacks Repeated login attempts to guess passwords

    4. Network Security Tools and Devices

    Tool/Device Function
    Firewall Blocks or permits traffic based on rules
    Antivirus/Antimalware Detects and removes malicious software
    Intrusion Detection System (IDS) Monitors traffic for suspicious activity
    Intrusion Prevention System (IPS) Blocks detected malicious activity
    VPN (Virtual Private Network) Encrypts data across public networks
    Proxy Server Acts as an intermediary for requests
    Access Control Lists (ACLs) Filter traffic on routers/switches

    5. Security Protocols

    Protocol Purpose
    SSL/TLS Encrypts data on websites (HTTPS)
    IPSec Secures IP communications (VPNs)
    WPA2/WPA3 Wireless network encryption
    SSH Secure remote access to devices
    HTTPS Secure web browsing

    6. User Authentication Methods

    Method Description
    Password-based Simple but vulnerable if weak
    Two-Factor Authentication (2FA) Combines password with code/token
    Biometric Uses fingerprint, face, iris, etc.
    Digital Certificates Used for encrypted communication and verification
    Tokens or Smart Cards Physical devices used for authentication

    7. Best Practices for Network Security
    * Use strong passwords and change them regularly.
    * Update software and firmware to patch vulnerabilities.
    * Segment the network using VLANs to contain breaches.
    * Limit user privileges using the principle of least privilege.
    * Use multi-factor authentication wherever possible.
    * Monitor logs and set up alerts for anomalies.
    * Encrypt sensitive data in transit and at rest.
    * Disable unused ports and services.
    * Educate users about social engineering attacks.
